Portaly 商品銷售 Webhook 自動記錄

自動化WebhookPortalyGoogle Sheets訂單記錄

這個工作流程自動處理 Portaly 平台的商品銷售 Webhook，智能提取訂單資訊並將其儲存到 Google Sheets 中。系統會接收來自 Portaly 的銷售通知，解析固定欄位資料（如訂單ID、客戶資訊、金額等）和自訂欄位資料，然後將所有資訊整理並記錄到 Google 試算表中，方便後續分析和管理。 主要功能： Webhook 接收：自動接收 Portaly 商品銷售通知 資料解析：智能提取訂單關鍵資訊和客戶自訂欄位 資料合併：將固定欄位和自訂欄位資料整合 Google Sheets 整合：自動記錄所有訂單資訊到試算表 即時更新：訂單發生時立即同步資料

設置說明

預先設定：需要設定 Portaly Webhook 和 Google Sheets API 存取權限

  1. Portaly Webhook 設定: 在 Portaly 平台設定 Webhook URL
    • 登入 Portaly 管理後台
    • 複製 n8n Webhook URL
    • 在 Portaly 中設定商品銷售 Webhook
    • 測試 Webhook 連接
  2. Google Sheets 設定: 準備用於記錄訂單資料的試算表
    • 複製提供的 Google Sheets 模板
    • 設定 OAuth 2.0 認證
    • 確認工作表名稱為 'Products'
    • 測試寫入權限
  3. 資料欄位設定: 確認 Google Sheets 包含所需欄位
    • 基本欄位：id, productId, customerEmail, customerName 等
    • 金額欄位：amount, discount, feeAmount, netTotal 等
    • 自訂欄位：n8n 使用經驗, 簡述問題或想討論的方向 等
    • 系統欄位：event, paymentMethod, createdAt, timestamp
