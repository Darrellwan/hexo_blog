Gmail 電子發票轉存 Google Sheet + AI 分析消費

自動化發票記帳GmailGoogle SheetsAI消費分析Slack

這個工作流程自動處理來自「財政部電子發票整合服務平台」的郵件，擷取電子發票資料並將其儲存到 Google Sheets 中，再使用 AI 進行消費分析。系統每天指定時間會監控 Gmail 中特定發件人的郵件，提取附件中的發票資訊，辨識「發票主資料(M)」和「商品明細(D)」，分別整理並存入兩個不同的工作表。最後，它會使用 xAI Grok 模型分析所有消費資料，生成包含摘要、類別統計、Top 5 店家和消費矩陣分析的深入報告，並將分析結果自動發送到 Slack 頻道，方便快速查看消費趨勢。 主要功能： 郵件監控：每日固定時間自動檢查財政部電子發票郵件 資料提取：智能解析附件內容取得發票資訊和商品明細 資料分類：自動區分「發票主資料(M)」和「商品明細(D)」 Google Sheets 整合：將不同類型資料存入對應工作表 AI 消費分析：使用 xAI Grok 模型生成多維度消費分析 Slack 通知：將完整分析報告即時發送到指定頻道

設置說明

預先設定：需要設定 Gmail、Google Sheets、AI 模型 (xAI Grok) 和 Slack 的 API 存取權限

  1. Gmail 設定: 設定 Gmail API 存取權限，用於讀取電子發票郵件
    • 設定 OAuth 2.0 認證
    • 授予郵件讀取權限
    • 設定郵件過濾條件為 'subject=財政部電子發票整合服務平台' 和 'from=einvoice@einvoice.nat.gov.tw'
  2. Google Sheets 設定: 準備用於儲存發票資料的試算表
    • 建立 Google 試算表
    • 設定「發票」工作表儲存主資料（載具名稱、載具號碼、發票日期、商店資訊、發票號碼、總金額、發票狀態）
    • 設定「細項」工作表儲存商品明細（品項ID、發票號碼、品項名稱、價格）
  3. AI 模型設定: 設定 xAI Grok 模型用於消費分析
    • 取得 xAI API 金鑰
    • 選擇 grok-3-fast-beta 模型
    • 設定分析提示詞以生成消費報告
  4. Slack 設定: 設定 Slack 整合用於發送分析結果
    • 建立 Slack App
    • 設定 Bot Token
    • 選擇目標頻道接收分析報告
