Line Message API 完整教學範例

自動化Line訊息機器人WebhookAPI教學範例多媒體訊息SocialChatbot

這個工作流程提供 Line Messaging API 的完整教學範例，包含自動回覆和主動推送兩種核心功能。當使用者透過 Line 傳送訊息時，系統會自動判斷訊息類型並回覆相同內容；同時也可以透過 Line UID 主動發送訊息給特定使用者。

功能特色
  • 接收來自 Line 的 Webhook 訊息
  • 條件判斷：僅處理 message 類型事件
  • 使用 replyToken 自動回覆相同內容給使用者
  • 支援主動推送訊息功能（需要 Line UID）
  • 完整的 Webhook 範例說明：文字、圖片、聲音、貼圖、表情符號、位置
  • 使用 HTTP Request 節點與 Line Messaging API 整合
  • 包含詳細的 JSON 格式範例和參數說明
設置說明

預先設定：需要設定 Line Developer Console 和取得 Channel Access Token

  1. Line Developer 設定: 在 Line Developer Console 建立 Channel 並設定 Webhook
    • 建立 Line Developer 帳號並登入
    • 建立新的 Messaging API Channel
    • 取得 Channel Access Token
    • 設定 Webhook URL 為 n8n 提供的 Webhook 端點
    • 開啟「Use webhook」選項
  2. 取得 Line Message API Token: 從 Line Developer Console 取得 Channel Access Token
    • 開啟 Line Developer Console (developers.line.biz/console/)
    • 選擇您的 Chatbot
    • 點擊 Messaging API 分頁
    • 複製 Channel Access Token
    • 參考工作流程中的教學圖示說明
  3. 認證設定: 在 n8n 中設定 Line API 認證
    • 建立 HTTP Bearer Auth 認證
    • 將 Channel Access Token 貼入認證欄位
    • 或使用 HTTP Header Auth：設定 Header Name 為 'Authorization'，Header Value 為 'Bearer YOUR_CHANNEL_ACCESS_TOKEN'
  4. 取得 Line UID: 取得自己或目標使用者的 Line UID
    • 開啟 Line Developer Console (developers.line.biz/console/)
    • 選擇您的 Chatbot
    • 點擊 Basic Settings 分頁
    • 複製 Your user ID
    • 參考工作流程中的教學圖示說明
  5. 推送訊息設定: 設定主動推送訊息功能
    • 取得目標使用者的 Line UID
    • 或取得群組的 Channel ID
    • 在 Edit Fields 節點中更新 line_uid 值
