Google nano banana 替換圖片

Google DriveAI 圖片生成Google Gemini

這個工作流程展示如何使用 Google Drive 觸發器配合 Google Gemini AI 模型進行自動圖片替換處理。當有新圖片上傳到指定的 Google Drive 資料夾時，系統會自動下載圖片，使用 Google Gemini 2.5 Flash Image Preview 模型進行「一個男人在凱旋門拿著原本產品」的圖片生成，然後將處理完成的圖片自動儲存到輸出資料夾。 主要功能： 觸發機制：Google Drive 資料夾監控，自動偵測新上傳的圖片檔案 檔案過濾：篩選符合條件的圖片格式進行處理 圖片下載：自動下載觸發的圖片檔案 AI 圖片處理：使用 Google Gemini 2.5 Flash Image Preview 模型生成新圖片 檔案轉換：將 AI 生成的結果轉換為可儲存的檔案格式 自動上傳：將處理完成的圖片自動儲存到指定的輸出資料夾

功能特色
  • 使用 Google Drive Trigger 監控指定資料夾的檔案變更
  • 透過 Filter 節點篩選符合條件的圖片檔案格式
  • 使用 Google Drive Download 節點下載觸發的原始圖片
  • 設定 AI Agent 節點進行圖片內容替換處理
  • 配置 Google Gemini Chat Model 節點使用 gemini-2.5-flash-image-preview 模型
  • 設定圖片生成提示詞：「一個男人在凱旋門拿著原本的產品，產生圖片給我」
  • 使用 Convert to File 節點將 AI 生成結果轉換為檔案格式
  • 透過 Google Drive Upload 節點將處理完成的圖片儲存到輸出資料夾
  • 自動命名檔案：使用時間戳記格式「yyyyMMdd_HHmmss.副檔名」
  • 完整自動化流程：從上傳到處理到儲存全程無需人工介入
設置說明

預先設定：需要 Google Drive API 存取權限和 Google Gemini API Key（建議使用付費方案）

  1. Google Drive API 設定: 建立和設定 Google Drive 整合
    • 建立 Google Cloud Console 專案
    • 啟用 Google Drive API
    • 建立 OAuth 2.0 認證憑證
    • 在 n8n 中設定 Google Drive OAuth2 API 連接
    • 建立輸入資料夾：「202508-Google-nano banana demo」
    • 建立輸出資料夾：「202508-Google-nano banana output」
  2. Google Gemini API 設定: 設定 Google Gemini AI 模型存取
    • 前往 Google AI Studio (aistudio.google.com)
    • 申請 Google Gemini API Key
    • 注意：免費方案可能無法使用，建議使用付費方案
    • 在 n8n 中建立 Google PaLM API 連接
    • 測試 gemini-2.5-flash-image-preview 模型連接
  3. 工作流程設定: 配置自動化流程參數
    • 設定 Google Drive Trigger 監控輸入資料夾
    • 調整 Filter 節點的檔案格式篩選條件
    • 自訂 AI Agent 的圖片生成提示詞
    • 設定檔案命名格式和輸出位置
    • 測試完整流程：上傳圖片 → AI 處理 → 自動儲存
