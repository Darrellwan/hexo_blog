LINE Messaging Community 節點範本

LINE聊天機器人自動回覆SocialChatbot

這個工作流程展示如何使用 LINE Messaging Community 節點建立 LINE Bot 自動回覆系統。包含接收訊息、事件分流、自動回覆，以及主動推播訊息給使用者的完整功能。適合用於客服機器人、自動通知系統或互動式 LINE 官方帳號。 主要功能： Webhook 接收：透過 LINE Messaging Trigger 接收使用者訊息和互動事件 智能分流：使用 Switch 節點區分 message 和 postback 事件 自動回覆：接收到訊息後自動回覆預設內容 主動推播：手動觸發推送訊息給指定使用者

功能特色
  • 接收 LINE Bot Webhook 事件（訊息、postback）
  • 使用 Switch 節點依據事件類型進行分流處理
  • message 事件會自動觸發回覆訊息
  • 支援 postback 事件處理（按鈕互動）
  • 手動觸發模式可主動推播訊息給特定使用者
  • 使用 Set 節點設定目標使用者的 LINE UID
  • 支援單一使用者精準推播功能
  • 包含完整的 LINE Messaging API 整合範例
設置說明

預先設定：需要申請 LINE Developers 帳號並建立 Messaging API Channel

  1. LINE Messaging API 設定: 在 LINE Developers Console 建立 Messaging API Channel
    • 前往 LINE Developers Console 建立新的 Provider
    • 建立 Messaging API Channel（選擇 Messaging API 類型）
    • 取得 Channel Secret 和 Channel Access Token
    • 在 n8n 中新增 Line Messaging API Credential
    • 設定 Webhook URL 為 n8n 提供的觸發網址
  2. 安裝 LINE Messaging Community 節點: 在 n8n 中安裝 @aotoki/n8n-nodes-line-messaging 社群節點
    • 進入 n8n Settings > Community Nodes
    • 搜尋並安裝 @aotoki/n8n-nodes-line-messaging
    • 重新啟動 n8n 以載入節點
    • 確認 Line Messaging Trigger 和 Line Messaging 節點可用
  3. 設定接收訊息流程: 配置自動回覆訊息的處理邏輯
    • 在 Line Messaging Trigger 節點中選擇要監聽的事件類型
    • 使用 Switch 節點設定分流條件（message / postback）
    • 在 Line Messaging-reply 節點中設定回覆文字
    • 使用 {{ $json.replyToken }} 確保正確回覆給發送者
    • 測試發送訊息給 LINE Bot 確認自動回覆功能
  4. 設定主動推播功能: 配置手動觸發推送訊息給特定使用者
    • 取得目標使用者的 LINE UID（可從 webhook 事件中取得）
    • 在「設定 Line 使用者資訊」節點中更新 line_uid
    • 修改「Line Messaging-送給單一使用者」節點的訊息內容
    • 手動執行 workflow 測試推播功能
    • 確認目標使用者收到推播訊息
